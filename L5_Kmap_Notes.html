① Key Definitions

Purpose:The purpose of a K-map is: To simplify a Boolean expression so the final circuit uses fewer gates, fewer inputs, less hardware, less power, and is easier to build.

Minterm: Product term that is 1 in exactly one row of the truth table.

Maxterm: Sum term that is 0 in exactly one row of the truth table.

Canonic Sum (SOP): Sum of minterms for which the function outputs 1.

F = Σm(minterms where output = 1)

Canonic Product (POS): Product of maxterms for which the function outputs 0.

F = ΠM(maxterms where output = 0)

K-Map: Graphical tool to simplify Boolean functions. Each cell = one minterm. Number of cells = 2ᴺ (N = variables).

③ Cell Adjacency Rules
  • Only 1 bit changes between adjacent cells (Gray code).
  • Adjacent = left, right, top, bottom neighbors.
  • Diagonal cells are NOT adjacent.
  • Top row ↔ Bottom row are adjacent (wrap-around).
  • Left column ↔ Right column are adjacent (wrap-around).
Grouping Rules (1's for SOP):
  1. Group size must be power of 2: 1, 2, 4, 8, 16…
  2. Each cell in group must be adjacent to at least one other cell in the group.
  3. Make groups as large as possible.
  4. Every 1 must be in at least one group.
  5. Groups can overlap.
  6. Wrap-around groups are allowed.
Larger group → fewer literals in term → simpler expression
④ SOP K-Map Minimization — Step by Step
Definition:

Why Do We Need SOP and POS?
A K-map represents a Boolean function. After grouping cells, we need a way to write the final answer. There are two common forms:

1
SOP (Sum of Products): OR together several AND terms, Group the 1s(A = 1, A'=0)
2
POS (Product of Sums):AND together several OR terms, Group the 0s(A = 0, A'=1)

Think of them as two different ways of describing the same circuit.

Procedure:
1
Map the expression: write truth table if needed, place 1 in each minterm cell.
2
Group all 1's into the largest possible power-of-2 groups (can overlap).
3
For each group, eliminate variables that appear both complemented and uncomplemented.
4
Write the minimum SOP: OR all group terms together.
Variable Elimination Rule:

A variable is kept if it appears in the same form in all cells of the group. It is eliminated if it appears as both X and X' within the group.

Group of 2 → eliminates 1 variable
Group of 4 → eliminates 2 variables
Group of 8 → eliminates 3 variables

Convert to standard SOP form first by expanding missing variables:

A'BCD' stays as is (already standard 4-var)

When you see:

F = Σm(1,3,5,7)

it means:

F = 1 at minterms 1, 3, 5, and 7

Every other row has:

F = 0

When you see:

F = ΠM(0,2,4,6)

it means:

F = 0 at rows 0, 2, 4, and 6

Every other row has:

F = 1

⑥ Don't Care Conditions

Don't cares = input combinations that never occur or whose output value doesn't matter. Marked as X in K-map.

  • When grouping 1's (SOP): treat X as 1 — include if it helps enlarge a group.
  • When grouping 0's (POS): treat X as 0 — include if it helps.
  • Only include X in a group if it simplifies the expression.
  • X cells that don't help → ignore them.

⑦ POS K-Map Minimization
1
Map the POS expression: place 0 in each maxterm cell.
2
Group all 0's (same rules as grouping 1's: power of 2, largest possible).
3
From each group, eliminate variables that change — keep those that are constant. Write a SUM term (OR). Variables appear uncomplemented if they are 0 in the group, complemented if they are 1.
4
AND all sum terms together → minimum POS.
Ex: Minimize (A+B+C)(A+B+C')(A+B'+C)(A+B'+C')(A'+B'+C)

⑧ SOP ↔ POS Conversion

One K-map holds both forms — cells with 1's give SOP; cells with 0's give POS.

  • To get minimum SOP: group the 1's.
  • To get minimum POS: group the 0's.
  • Use conversion to find which form needs fewer gates.

Key Takeaways
✓
K-map rows/columns must follow Gray code — only 1 bit change between adjacent cells. This is non-negotiable!
✓
Always form the largest possible groups (power of 2). Bigger groups → fewer literals → simpler circuit.
✓
Wrap-around adjacency: top↔bottom row and left↔right column are adjacent. Corner cells can form a group of 4!
✓
SOP → group 1's; POS → group 0's. Both derive minimum expressions from the same K-map.
✓
Don't cares (X): treat as 1 for SOP grouping or 0 for POS grouping — only if they allow a larger group.
✓
After minimization, compare SOP vs POS gate count to choose the most economical implementation.
