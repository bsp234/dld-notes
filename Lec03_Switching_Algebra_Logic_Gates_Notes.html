① Boolean Algebra

Boolean Algebra is the mathematical tool that describes the relationship between a logic circuit's inputs and outputs as an algebraic expression.

  • Only two values: 0 and 1
  • Logic 0 = false, off, low, no, open switch
  • Logic 1 = true, on, high, yes, closed switch
  • Three basic operations: OR, AND, NOT
  • Corresponds to: addition (+), multiplication (·), complement (‾)
Uses of Boolean Algebra

Describing circuits • Analyzing behavior • Designing new circuits • Simplifying gate count • Implementing digital systems

⑩ Universal Gates: NAND & NOR

Both NAND and NOR can implement any Boolean function alone:

NAND → NOT, AND, OR:
  • NOT: NAND with inputs tied together: AA' = Ā·Ā = Ā
  • AND: NAND + NOT: (AB)'' = AB
  • OR: NAND each input then NAND: Ā·B̄ = A+B (DeMorgan)
NOR → NOT, OR, AND:
  • NOT: NOR with inputs tied together
  • OR: NOR + NOT
  • AND: NOR each input then NOR

③ Commutative & Associative
Commutative Law

Order of variables does NOT matter:

A + B = B + A
A · B = B · A
Associative Law

Grouping of variables does NOT matter:

A+(B+C) = (A+B)+C
A·(B·C) = (A·B)·C

These allow us to reorder and regroup terms freely when simplifying Boolean expressions.

④ Distributive Law

AND distributes over OR (like regular algebra):

A(B + C) = AB + AC

OR also distributes over AND (unlike regular algebra!):

A + (B·C) = (A+B)·(A+C)
Simplification Example

F = X̄YZ + X̄YZ̄ + XZ
= X̄Y(Z+Z̄) + XZ    [distrib]
= X̄Y·1 + XZ        [R6]
= X̄Y + XZ         [R4] ✓

Result: 3 AND gates + OR reduced to 2 AND gates + OR = fewer gates!

⑤ De Morgan's Theorems

Two fundamental theorems relating complements of OR and AND:

$\overline{A+B} = \bar{A} \cdot \bar{B}$

NOR = AND of complements

$\overline{A \cdot B} = \bar{A} + \bar{B}$

NAND = OR of complements

🔑 How to apply De Morgan
  1. Complement entire expression
  2. Change each AND (·) to OR (+) and vice versa
  3. Complement each individual variable
Example

$\overline{ABC(\overline{A+D})}$ with A=0,B=1,C=1,D=1
= Ā·B·C·$\overline{(A+D)}$ = 1·1·1·$\overline{1}$ = 1·0 = 0

⑦ Evaluating Boolean Expressions

Order of operations (like PEMDAS in regular math):

1
Invert single variables with complement bars first
2
Evaluate parenthesized expressions
3
Perform AND (·) operations before OR (+) — unless parentheses say otherwise
4
For a bar over a group, evaluate inside first, then complement the result
Note: A+B·C ≠ (A+B)·C

Without parentheses, AND is always evaluated first!

✅ Key Takeaways
✓
N-input truth table has 2N rows. AND=all-1, OR=any-1, NOT=flip.
✓
NAND and NOR are universal — any circuit can be built from NAND gates alone (or NOR alone).
✓
XOR outputs 1 when inputs differ. XNOR outputs 1 when inputs are the same.
✓
De Morgan: complement a sum → product of complements; complement a product → sum of complements.
✓
Simplifying Boolean expressions reduces gate count → smaller, faster, cheaper circuits.

Ex 2: x = (A+B)·C

OR gate (A,B) → output (A+B) → feeds AND gate with C → final output x. Parentheses force OR first here.
