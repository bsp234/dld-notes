① What is a Register?

A flip-flop stores one bit of information. When n flip-flops are grouped to store n bits (e.g. an n-bit number), the group is called a register.

  • Two basic functions: data storage & data movement
  • A common clock drives all flip-flops in the register
  • A register with shifting ability = Shift Register
n flip-flops → n-bit register (stores 2ⁿ states)
Data Transfer Modes

Parallel Transfer: all n bits sent at once using n separate wires — fast!

Serial Transfer: all bits sent one at a time over a single wire, taking n clock cycles — slower but uses fewer wires

② Shift Register Types
  • SISO Serial In — Serial Out
  • SIPO Serial In — Parallel Out
  • PISO Parallel In — Serial Out
  • PIPO Parallel In — Parallel Out

Universal: supports all four modes (e.g. 74LS194)

Common Tasks

Serial/Parallel data conversion · UART · Time delay · Ring counter · Johnson counter · Memory device

⑤ PIPO — Parallel In Parallel Out

All bits loaded at once AND all bits appear at output simultaneously. Fastest mode — used as a simple storage buffer.

One clock pulse → all bits stored and available at output!

⑥ Universal Shift Register

Has both serial and parallel input and output capability — supports all four modes.

  • Examples: 74LS194, 74LS195, 74HC194, CMOS 4035
  • Available as 4-bit multi-function devices
  • Bi-directional shifting (left and right)
Why Universal?

Used in arithmetic operations — shifting left multiplies by 2, shifting right divides by 2. Eliminates need for separate serial/parallel registers.

  • n-bit ring counter = mod-n counter
  • 4-bit → only 4 of 16 possible states used (inefficient!)
  • Advantage: Self-decoding — no extra decode circuit needed
⑧ Johnson Counter

Variation of ring counter where the inverted output of the last stage is fed back to the first stage input. Also called Twisted-Ring Counter.

⑨ Counter Types
Asynchronous (Ripple)
  • Flip-flops do NOT share a common clock
  • Each FF clocked by previous FF output
  • Changes propagate with a ripple delay
  • Simpler design but slower
Synchronous
  • All flip-flops share the same clock
  • All FFs change state simultaneously
  • No ripple delay — faster & more predictable
  • More complex design
Shift Register vs Binary Counter

Shift register counters (ring/Johnson) use feedback from serial output → serial input to produce special count sequences. Much simpler decoding than binary counters!

Key Takeaways
✓
A register = n flip-flops with a common clock. Two roles: storage and movement
✓
Four shift register types: SISO, SIPO, PISO, PIPO. Universal SR supports all four modes
✓
Ring counter: feedback Q_last → D_first. n-bit = mod-n. Self-decoding but state-inefficient
✓
Johnson counter: feedback Q̄_last → D_first. n-bit = mod-2n. More efficient than ring counter
✓
Async counters have ripple delay (no common clock). Sync counters use one clock — all FFs change together
