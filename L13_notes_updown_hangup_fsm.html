① Up Counters (Single-Mode)

A counter is single-mode if: the only external input is the clock, and the only outputs are the flip-flop states.

Count Sequences

2-bit: 00 → 01 → 10 → 11 → 00
3-bit: 000 → 001 → 010 → 011 → 100 → 101 → 110 → 111 → 000
4-bit: 0000 → … → 1111 → 0000

4-bit Decade Counter (mod-10)

Counts: 0000 → 0001 → … → 1001 → 0000
Skips states 1010–1111 (unused!)
Used in BCD applications

② Down Counters (Single-Mode)

Counts downward with each clock pulse. Still a single-mode counter (only clock as input).

4-bit Down Sequence

1111 → 1110 → 1101 → … → 0001 → 0000 → 1111

  • Designed same way as up counter (7 steps)
  • Next-state table counts in reverse order
  • JK/D inputs derived by same procedure

③ Up/Down Counter (Multi-Mode)

Can count in either direction — also called a bidirectional counter.

Multi-mode = has additional external inputs besides clock
  • Control input UP/DOWN selects direction
  • UP/DOWN = 1 → counts up
  • UP/DOWN = 0 → counts down
3-bit Up/Down Counter Logic

Each T flip-flop input depends on both the current state AND the UP/DOWN control signal. Two separate next-state sequences are merged into one excitation table using the control variable.

④ Enable & Clear Control
Enable Control
  • Allows counting to be inhibited (paused)
  • If Enable = 0: all T inputs = 0 → counter holds present state
  • If Enable = 1: counter counts normally
T_i = Enable · (counter logic)
Clear Control
  • Resets all flip-flops to 0
  • Asynchronous clear: happens immediately
  • Synchronous clear: happens on next clock edge

⑤ Hang-Up States ⚠️

What happens if a ring/Johnson counter enters an unused state?

Safe Recovery

If the next state is a valid state → counter eventually returns to the counting sequence. Nothing harmful happens.

Good: unused → valid → sequence

Unused state next→ valid state → normal counting resumes ✓

Hang-Up (Defective!)

If unused states keep transitioning to each other → counter is stuck forever — never returns to valid counting sequence!

⚠️ 3-bit Johnson Hang-up

Unused states: 010 and 101
If enters 010 → next = 101
If enters 101 → next = 010
Cycles between 010↔101 forever!
Counter is worthless in this condition.

Fix: Self-Starting Design

Design the counter so that ALL unused states transition to a valid state on the next clock pulse. This ensures the counter always recovers — no hang-up possible.

⑥ Finite State Machine (FSM)

A mathematical model of computation used to design both computer programs and sequential logic circuits.

  • Machine is in one state at a time (current state)
  • Transitions happen on triggering events/conditions
  • Defined by: list of states + triggering conditions for transitions
FSM = States + Inputs + Transitions + Outputs

Key Takeaways — L13
✓
Single-mode counters (up/down): clock is the only external input. Outputs = FF states only
✓
Multi-mode counters have additional control inputs (e.g. up/down direction, enable, clear)
✓
Enable=0 freezes counter (all T inputs = 0). Clear resets all FFs to 0
✓
Hang-up state: unused state that loops to itself or other unused states → counter stuck permanently
✓
FSM = abstract model with finite states, transitions on events. Counters are a special type of FSM
✓
Fix hang-ups by self-starting design: every unused state must have a valid next state in the sequence
