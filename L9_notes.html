① Comparators — Overview
  • Arithmetic circuit that compares two binary numbers A and B
  • Produces three outputs: AeqB, AgtB, AltB
  • AeqB = 1 if A = B
  • AgtB = 1 if A > B
  • AltB = 1 if A < B

For 4-bit numbers: A = a₃a₂a₁a₀, B = b₃b₂b₁b₀

Define intermediate signals iₖ — each iₖ = 1 if aₖ = bₖ:

iₖ = aₖ XNOR bₖ = ākb̄k + aₖbₖ
AeqB = i₃ · i₂ · i₁ · i₀

AgtB — check bits from MSB down. First bit-position k where aₖ ≠ bₖ decides:

AgtB = a₃b̄₃ + i₃a₂b̄₂ + i₃i₂a₁b̄₁ + i₃i₂i₁a₀b̄₀
AltB = 1 − AeqB − AgtB
💡 Key Idea

Truth table approach works but gets huge for large n. Pair-by-pair approach is much more efficient!

Intermediate Signals (iₖ)
aₖbₖiₖ = aₖ XNOR bₖMeaning
001equal
010differ
100differ
111equal

iₖ = 1 means the two bits at position k are the same

AgtB — Derivation Logic
  1. Compare MSB first (bit 3). If a₃=1, b₃=0 → A > B immediately
  2. If i₃=1 (MSB equal), check next bit (bit 2)
  3. If i₃·i₂=1, check bit 1, and so on…
  4. Each term in AgtB covers a new "first-difference" scenario
Circuit implementation uses:

XNOR gates → iₖ signals
AND gates → chain equality checks
OR gate → combine all AgtB terms

② NAND Gate

NAND = AND output complemented

Y = ̄(x₁ · x₂)

DeMorgan's: NAND of x₁ and x₂ is equivalent to ORing their complements:

x̄₁ · x̄₂ → x̄₁ + x̄₂

Universal gate — any logic function can be built using only NAND gates.

Any SOP (Sum of Products) expression → implementable with only NANDs using double inversion:

f = x₁x₂ + x₃x₄x₅

Apply double inversion → two-level NAND-NAND network

③ NOR Gate

NOR = OR output complemented

Y = ̄(x₁ + x₂)

DeMorgan's: NOR function = first invert inputs, then AND them:

x̄₁ + x̄₂ → x̄₁ · x̄₂

Universal gate — any logic function can be built using only NOR gates.

Any POS (Product of Sums) expression → implementable with only NORs:

f = (x₁+x₂)(x₃+x₄+x₅)

Apply double inversion → two-level NOR-NOR network

⚠️ Double Inversion Trick

To implement any expression with only one gate type:

  1. Write SOP (for NAND) or POS (for NOR)
  2. Apply double inversion (adds two bubbles)
  3. Bubbles cancel on shared wires → pure NAND or NOR network
④ Baugh-Wooley Multiplier

A method for multiplying two's complement signed binary numbers using only AND gates and adders — no sign correction needed afterward.

Example 1: 10 × 12 = 120

1010 × 1100
= 1111000₂ = 120₁₀ ✓

Example 2: 9 × 6 = 54

0110 × 1001
= 110110₂ = 54₁₀ ✓

Why Baugh-Wooley?
  • Handles signed numbers directly
  • Uses full adder arrays — regular structure
  • More hardware-efficient than sign-extension method
⑤ Logic Synthesis Using Multiplexers

MUXes can implement any logic function by connecting inputs to fixed 0/1 or variable signals.

Ex 1: XOR using 4-to-1 MUX (F = w₁ ⊕ w₂)

Use w₁,w₂ as select lines:

I₀=0, I₁=1
I₂=1, I₃=0
→ Connect directly!

Ex 2: XOR using 2-to-1 MUX (F = w₁ ⊕ w₂)

Use w₁ as select line:
• When w₁=0 → F = w₂   (I₀ = w₂)
• When w₁=1 → F = w̄₂  (I₁ = w̄₂)
→ One 2-to-1 MUX + one inverter!

Ex 3: 3-input Majority using 4-to-1 MUX

F = 1 if majority of w₁,w₂,w₃ = 1
Use w₁,w₂ as selects:
I₀=0, I₁=w₃, I₂=w₃, I₃=1

⑥ Priority Encoder
  • Each input has an associated priority level
  • Outputs indicate the highest-priority active input
  • Lower-priority inputs are ignored when a higher one is asserted

Outputs y₁, y₀ → binary number of highest priority active input

Output z → validity flag

z = 1 if any input = 1
z = 0 if all inputs = 0

4-input Priority Encoder truth table:

w₃w₂w₁w₀y₁y₀z
0000dd0
0001001
001d011
01dd101
1ddd111

d = don't care (overridden by higher priority)

✓ Key Takeaways
✓
Comparator uses XNOR (iₖ) to detect equality bit-by-bit. AeqB = AND of all iₖ
✓
AgtB built from first differing bit — check MSB first, chain downward
✓
NAND & NOR are universal — any function implementable with either one alone
✓
SOP → NAND-NAND  |  POS → NOR-NOR  |  Use double inversion
✓
MUX can synthesize any function: connect fixed 0/1 or variable inputs to data lines
✓
