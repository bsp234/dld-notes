FeedbackNoneHas feedback

Sequential circuits have one or more outputs fed back to inputs. This feedback creates memory — the circuit "remembers" past states.

Real-World Examples:
Traffic Lights Clocks Timers Counters Registers
Multivibrators:
  • Astable: No stable state — oscillates (clock generator)
  • Monostable: One stable state — pulse generator
  • Bistable: Two stable states → LATCH / FLIP-FLOP
② Latches Overview

Latch: Bistable memory element. Has two stable states (0 or 1). Does NOT need a clock — changes state directly with inputs.

④ Gated SR & D Latches
Gated SR Latch

Adds an Enable (EN/CLK) input. Latch only responds when EN=1.

NOR-based: S,R active high. NAND-based: S,R active low through AND gates at EN.

Gated D Latch

D (Data/Delay) latch: single data input. Eliminates the forbidden state by tying S = D and R = D'.

Used as data register, data storage, pipeline stage.

⑤ Flip-Flops — Edge Triggered

Flip-Flop: Changes state only at the clock edge — not during the entire clock HIGH period like a latch.

↑ Positive Edge (Rising) — clock input has no bubble
↓ Negative Edge (Falling) — clock input has a bubble (∘)

⑥ Master-Slave D FF & T FF
Master-Slave D Flip-Flop

Two Gated D latches in series. Solves the transparency problem.

CLKMaster (Qm)Slave (Qs)
= 1 (HIGH)Tracks D inputLocked (holds)
= 0 (LOW)Locked (holds)Tracks Qm

Result: Output Qs only updates when CLK transitions 1→0. Clean edge behavior!

T (Toggle) Flip-Flop

T = Toggle. Built with a D flip-flop and XOR gate: D = T⊕Q.

TQ(next)Action
0QHold (no change)
1Q'Toggle!
Q(next) = T⊕Q

Used in counters — toggles on every clock if T=1.

⑦ JK Flip-Flop

Most versatile FF. Solves the forbidden state of SR FF — J=K=1 now toggles the output!

⑧ Operating Characteristics
① Propagation Delay (tₚ)

Time from clock edge to output change.

SymbolDirection
tPLHOutput: LOW → HIGH
tPHLOutput: HIGH → LOW
② Setup Time (tₛ)

Minimum time inputs must be stable BEFORE the triggering clock edge. Input must arrive ≥ tₛ before clock edge.

Data must be stable for tₛ BEFORE ↑ clock edge
③ Hold Time (tₕ)

Minimum time inputs must remain stable AFTER the triggering clock edge.

Data must remain stable for tₕ AFTER ↑ clock edge
Other Parameters:
  • Maximum clock frequency: fmax = 1/(tₛ + tₕ + tₚ)
  • Pulse widths: minimum HIGH/LOW times
  • Power dissipation

Key Takeaways
✓
Sequential circuits have memory via feedback. Output depends on inputs AND current state.
✓
Latches are level-sensitive (change while EN=1). Flip-flops are edge-sensitive (change at clock edge only).
✓
SR FF forbidden state (S=R=1) solved by JK FF (toggles instead).
✓
Master-Slave FF: master samples while CLK=1, slave outputs when CLK=0. Clean edge-triggered behavior.
✓
Setup/Hold time violations cause unreliable operation (metastability). Always meet timing specs!
