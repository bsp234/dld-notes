① Multiplexer (MUX)

A digital switch — selects one of many input data sources and routes it to a single output channel.

Many → One
Multiple inputs
Single output
Selector Bits
n select lines
→ 2ⁿ inputs
Also called
Data Selector
  • Output X / Z depends entirely on the select input code
  • Performs multiplexing — many inputs, one output
  • Common types: 2-to-1, 4-to-1, 8-to-1, 16-to-1

② 2-to-1 MUX

1 select line (S), 2 inputs (I₀, I₁), 1 output Z. S chooses which input passes to output.

Boolean Equation
Z = S̄·I₀ + S·I₁

S=1 → passes I₁

③ 4-to-1 MUX

2 select lines (S₁S₀), 4 inputs (I₀–I₃), 1 output Z.

Select Table
S₁S₀Output Z
00I₀
01I₁
10I₂

S₁S₀ = binary address of the input to select
Boolean Equation
Z = S̄₁S̄₀I₀ + S̄₁S₀I₁ + S₁S̄₀I₂ + S₁S₀I₃
Gate implementation:
  • 4 AND gates (3-input each)
  • 1 OR gate (4-input)
  • 2 NOT gates (for S̄₁, S̄₀)

n select lines → 2ⁿ inputs

⑤ Building Larger MUX from Smaller
🔧 Class Exercise 1: 4-to-1 from three 2-to-1 MUXes
Stage 1 (S₀ selects):
  MUX-A: inputs I₀, I₁ → out A  (sel = S₀)
  MUX-B: inputs I₂, I₃ → out B  (sel = S₀)
Stage 2 (S₁ selects):
  MUX-C: inputs A, B   → Z      (sel = S₁)
🔧 Class Exercise 2: 8-to-1 from two 4-to-1 + one 2-to-1
MUX-A (4-to-1): I₀–I₃, sel = S₁S₀ → out A
MUX-B (4-to-1): I₄–I₇, sel = S₁S₀ → out B
MUX-C (2-to-1): inputs A, B, sel = S₂ → Z
(S₂ = MSB chooses which 4-to-1 block)

⑥ Demultiplexer (DEMUX)

Reverse of MUX — takes one input and routes it to one of several outputs based on select lines.

One → Many
Single input I₀
Multiple outputs Q
n Select Lines
→ 2ⁿ outputs
Only ONE active
  • 1 data-input line (I₀)
  • N data-output lines (Q₀, Q₁ … Q_{N-1})
  • Select inputs (S₀, S₁ … S_{n-1}) choose active output

⑨ Code Converters

Convert digital data from one coding scheme to another. Most decoders ARE code converters (e.g. BCD-to-7seg).

Binary → Gray Code
Rule: MSB unchanged, each other bit = XOR with previous binary bit
G₁ = B₁  (MSB, unchanged)
G₂ = B₁ ⊕ B₂
G₃ = B₂ ⊕ B₃
G₄ = B₃ ⊕ B₄
Gray → Binary Code
Rule: MSB unchanged, each other bit = XOR with previous binary result
B₁ = G₁  (MSB, unchanged)
B₂ = B₁ ⊕ G₂
B₃ = B₂ ⊕ G₃
B₄ = B₃ ⊕ G₄
Ex: Binary 1011 → Gray
B: 1–0–1–1
G₁ = 1
G₂ = 1⊕0 = 1
G₃ = 0⊕1 = 1
G₄ = 1⊕1 = 0
Gray = 1110
Ex: Gray 1110 → Binary
G: 1–1–1–0
B₁ = 1
B₂ = 1⊕1 = 0
B₃ = 0⊕1 = 1
B₄ = 1⊕0 = 1
Binary = 1011 ✓

Key Takeaways
✓
MUX = Data Selector: n select lines → 2ⁿ inputs, routes ONE input to the output. Z = Σ(minterm · Iₙ)
✓
2-to-1 MUX equation: Z = S̄I₀ + SI₁. Each AND term = minterm × data input.
✓
Larger MUX from smaller: two 4-to-1 + one 2-to-1 = 8-to-1. MSB selects between blocks.
✓
DEMUX = Data Distributor: reverses MUX. One input → selected output. Qᵢ = (minterm of select) · I₀
✓
1-to-4 DEMUX: Q₀=S̄₁S̄₀I₀, Q₁=S̄₁S₀I₀, Q₂=S₁S̄₀I₀, Q₃=S₁S₀I₀
✓
Binary→Gray: MSB stays; each Gₙ = Bₙ₋₁ ⊕ Bₙ (XOR adjacent binary bits)
✓
Gray→Binary: MSB stays; each Bₙ = Bₙ₋₁ ⊕ Gₙ (XOR previous binary result with Gray bit)
Quick Reference Formulas
2-to-1 MUX: Z = S̄I₀ + SI₁
4-to-1 MUX: Z = S̄₁S̄₀I₀ + S̄₁S₀I₁ + S₁S̄₀I₂ + S₁S₀I₃
1-to-2 DEMUX: Q₀=S̄I₀, Q₁=SI₀
Bin→Gray: G₁=B₁; Gₙ=Bₙ₋₁⊕Bₙ (n≥2)
Gray→Bin: B₁=G₁; Bₙ=Bₙ₋₁⊕Gₙ (n≥2)
